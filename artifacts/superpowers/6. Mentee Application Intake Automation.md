# 6. Mentee Application Intake Automation

## Plain-English Goal

When a mentee submits the application form, the system should immediately prepare the row for internal review.

This is not the same as final selection scoring.

Intake answers this question:

```text
Is this application clean enough for review, or does the team need to check something first?
```

## Scope

Version 1 is for mentees only.

Mentor intake can use the same pattern later, but the first build should stay focused on the mentee application sheet.

## Where This Lives

This automation lives in the existing mentee response spreadsheet:

```text
YDP Mentees Application (Cohort 2) (Responses)
```

It adds columns to the same form response tab, at the far right. It does not create a new response sheet.

It also creates one helper tab:

```text
Intake Dictionary
```

## What Happens On Form Submit

When a new mentee application arrives:

1. The applicant receives the normal acknowledgement email.
2. The row gets or reuses a stable `Mentee ID`.
3. The row is checked for duplicate email and phone.
4. The row is checked for required-field completeness.
5. The row gets an intake score and status.
6. The row gets internal notes explaining what happened.

## New Intake Columns

| Column | Meaning |
| --- | --- |
| `Intake Score` | Rule-based score out of `100`. This is a quick triage score, not the final selection score. |
| `Intake Status` | Internal review status: `Ready for Review`, `Needs Review`, or `Duplicate Review`. |
| `Intake Recommendation` | Human-readable next action for the team. |
| `Intake Notes` | Short explanation of what is missing or what duplicate was found. |
| `Duplicate Match Type` | Shows whether duplicate was found by `Email`, `Phone`, `Email + Phone`, or `None`. |
| `Intake Processed At` | Timestamp when the automation processed the row. |

## Intake Status Values

| Status | Meaning | What the team should do |
| --- | --- | --- |
| `Ready for Review` | The application is complete enough and no duplicate email/phone was found. | Review normally. |
| `Needs Review` | Important fields are missing or weak. | Check the notes before deciding. |
| `Duplicate Review` | Email or phone already exists on another row. | Compare with the original submission before deciding. |

## Intake Score

The intake score is rule-based. It does not use Gemini.

| Area | Points | Plain meaning |
| --- | ---: | --- |
| Required fields complete | 40 | Email, name, phone, location, career path, goals, and commitment fields are filled. |
| Career path clarity | 15 | The applicant selected or clearly stated a data career path. |
| Learning effort | 20 | The applicant has taken courses, practiced, built something, or shown effort. |
| Goals quality | 15 | The goals are meaningful enough for a mentor to work with. |
| Commitment readiness | 10 | The applicant agreed to commit and gave reasonable weekly availability. |
| Total | 100 | Quick intake score only. |

## How This Differs From Gemini Selection Scoring

| Layer | Where it lives | Uses Gemini? | Purpose |
| --- | --- | --- | --- |
| Intake score | Mentee response sheet | No | Fast form-submit triage and data hygiene. |
| Selection score | Matching workbook | Yes | Deeper review for selection and matching. |

The intake score is not used to reject anyone automatically.

## Duplicate Rules

The duplicate check should use:

- email address,
- phone number.

Duplicate applicants still receive the normal acknowledgement email. Duplicate handling is internal only.

## Intake Dictionary Tab

The `Intake Dictionary` tab should explain:

- each intake column,
- possible values,
- score breakdown,
- operating instructions,
- what to do for duplicate/incomplete/ready rows.

## Relationship To Matching

The matching workbook can still sync from the mentee response sheet.

Running `YDP Matching > Sync source snapshots from forms` only refreshes:

- `Mentee Source Snapshot`,
- `Mentor Source Snapshot`.

It does not delete:

- `Mentee Scores`,
- `Match Recommendations`,
- `Matched Pairs`.

So existing Gemini scoring work in the matching workbook should not be lost by syncing source snapshots.

## Mentor Scoring Note

Mentors should not be scored the same way as mentees.

For mentors, the better concept is a `Mentor Profile` or `Mentor Capacity/Fit Summary`, not a pass/fail score. The matching system should use mentor:

- expertise,
- current role,
- years of experience,
- availability,
- mentee capacity,
- preferred mentee background,
- communication preference.

Gemini can help summarize why a mentor is a good match, but mentor matching should mainly respect expertise and capacity.
