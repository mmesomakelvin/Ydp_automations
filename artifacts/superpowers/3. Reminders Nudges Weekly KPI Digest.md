# 1. Reminders, Nudges, and Weekly KPI Digest Superpower

## Plain-English Goal

This automation helps the YDP Mentorship Team keep the program moving without manually chasing everyone.

It should remind people before important activities, follow up when someone misses something, and send the team a weekly Monday summary of how the program is going.

## What We Are Building

We are building an Apps Script automation that reads live program records from Google Sheets and sends messages at the right time.

The core parts are:

1. **Reminders**
   - Remind mentees about monthly check-in/workshop sessions.
   - Remind mentees about monthly guest speaker sessions.
   - Remind mentees to complete monthly feedback/project update forms.

2. **No-show nudges**
   - Check attendance after a session or workshop.
   - Detect who did not attend.
   - Send them a polite follow-up message.

3. **Weekly KPI digest**
   - Every Monday, send stakeholders a short summary of program health.
   - Pull numbers from Sheets or a dashboard.
   - Highlight important issues like poor attendance, missed sessions, or delayed matching.

## Why This Matters

Without this automation, the team has to manually:

- remember every session and deadline,
- send reminders one by one,
- track who missed what,
- chase no-shows,
- prepare weekly updates for stakeholders.

With this automation, the system does the repetitive follow-up work while the team focuses on decisions and support.

## Message Channels

### Email

This is the easiest and most reliable first version.

Apps Script can send emails directly using `MailApp`.

Decision: **Version 1 will use email only.**

### WhatsApp

WhatsApp needs an external provider or a semi-manual workflow.

Options:

1. **Manual WhatsApp links**
   - Apps Script creates pre-filled WhatsApp message links.
   - A team member clicks and sends manually.
   - Easiest, but not fully automatic.

2. **WhatsApp Cloud API**
   - Fully automated WhatsApp messages.
   - Needs Meta Business setup, approved templates, phone number setup, and API credentials.

3. **Twilio WhatsApp**
   - Fully automated WhatsApp messages through Twilio.
   - Easier developer experience, but has cost and setup requirements.

## Version 1 Program Calendar Scope

The Program Calendar will focus on monthly mentee activities, not individual mentor-mentee 1:1 sessions yet.

The first version should support:

1. **Monthly mentee check-in/workshop**
   - A recurring monthly meeting with mentees.
   - Used to understand how they are doing in the program.

2. **Monthly guest speaker session**
   - A one-hour session with an invited guest speaker.
   - Can use the same reminder system as the monthly workshop.

3. **Monthly feedback/project update form**
   - Mentees fill a monthly form.
   - The form asks what they have learned, what projects they are working on, and how they rate their assigned mentor.

Individual 1:1 mentor-mentee session reminders can be added later if needed.

## Features We May Discard

These are optional and should only be built if they are truly needed:

- Fully automatic WhatsApp sending.
- Multiple reminder times for every event.
- Complex risk scoring.
- Looker Studio automation if a Google Sheets dashboard is enough.
- Stakeholder digest with long narrative analysis.
- SMS reminders.

## First Version Recommendation

Build a simple, reliable version first:

1. Email reminders for monthly check-ins/workshops, guest speaker sessions, and feedback form deadlines.
2. Email nudges for missed attendance.
3. Monday email digest from Sheets.
4. Manual WhatsApp links as a helper, not full WhatsApp automation.

Then add fully automated WhatsApp only if the team really needs it and has the required WhatsApp API setup.

## Brainstorming Decisions So Far

These are the decisions confirmed during the Superpowers brainstorming session.

### Scope

- Version 1 is email-only.
- WhatsApp automation is deferred because it requires WhatsApp Cloud API, Twilio, or another provider.
- The first version focuses on monthly mentee program activities, not individual mentor-mentee 1:1 meetings.
- The automation should support:
  - monthly mentee check-in/workshop reminders,
  - monthly guest speaker session reminders,
  - monthly feedback/project update form reminders,
  - no-show nudges,
  - stakeholder KPI report after meaningful activity.

### Reminder Recipients

Reminders depend on event type:

- Monthly check-in/workshop: mentees receive reminders; mentors only if they are expected to attend.
- Guest speaker session: mentees receive reminders; mentors only if needed.
- Monthly feedback/project update form: mentees only.

### Reminder Timing

Live monthly sessions should have three reminder points:

- 3 days before.
- 24 hours before.
- Morning of the event.

Monthly feedback/project update forms should have three reminder points:

- 7 days before the deadline.
- 3 days before the deadline.
- Deadline day.

### Attendance And No-Show Rules

- Attendance tracking is for mentees only in version 1.
- Attendance will use one reusable Google Attendance Form.
- The attendance form should collect:
  - `Timestamp`
  - `Email Address`
  - `Full Name`
  - `Session/Event`
- A no-show means a mentee did not fill the attendance form.
- No-show nudges go out the next morning.
- No-show nudges go to the participant only.
- A mentee becomes at-risk after 2 missed monthly sessions.

### Feedback Form Rules

- The monthly feedback form is for mentees only.
- Use one reusable Google Form every month.
- The form should include a `Month` question.
- The form should collect what mentees learned, what projects they worked on, blockers/progress, and mentor rating/review.
- If a mentee does not submit the form:
  - send a nudge after the deadline,
  - include the missing submission in the stakeholder report metrics.
- Post-deadline feedback nudges go out:
  - next morning after the deadline,
  - again 2 days later if still not submitted.

### Stakeholder Report Rules

- The stakeholder report should be detailed enough for operations, but privacy-safe.
- Recipients: program team and YDP leadership.
- The report should send only after a workshop, guest speaker session, or feedback form deadline happened.
- It should not send every Monday if nothing meaningful happened.
- The report should include:
  - KPI metrics,
  - chart snapshots inside the email,
  - Gemini-assisted summary write-up,
  - PDF attachment,
  - link to the live Google Sheets dashboard.
- The PDF should be generated only when a report is sent.
- No individual mentee names should appear in the email, PDF, or dashboard charts.
- At-risk reporting should include:
  - total at-risk count,
  - reason breakdown,
  - anonymous mentee IDs.
- The automation should create anonymous IDs like `Mentee-001`, `Mentee-002`, and store them in the `Matched Pairs` sheet.
- Report tone should be formal and operational.

### Dashboard Decision

- Start with a Google Sheets dashboard.
- Move to Looker Studio/Data Studio later only if needed.
- Reason: Google Sheets is easier to automate, export as PDF, and use for embedded chart snapshots in email.

### Active Mentee Source

- The automation should use the `Matched Pairs` sheet as the source of active mentees.
- The `Matched Pairs` sheet should support full operations tracking.

Recommended minimum columns:

| Header |
| --- |
| Anonymous Mentee ID |
| Mentee Name |
| Mentee Email |
| Mentor Name |
| Mentor Email |
| Match Status |
| Track |
| Start Date |
| Active Status |
| Last Check-in Date |
| Risk Status |
| Missed Sessions Count |
| Feedback Completion Count |
| Mentor Rating Average |

### KPI Set For Version 1

Version 1 should include core engagement and learning KPIs:

- Attendance rate.
- No-show count.
- Feedback completion rate.
- At-risk mentee count.
- Projects submitted.
- Learning updates submitted.
- Average mentor rating.

Version 1 will not include:

- complex mentor performance trends,
- cohort risk scoring beyond the agreed at-risk rules,
- month-over-month comparisons unless added later.

### Gemini Role

Recommended approach: Gemini-assisted reporting.

The script and Sheets should calculate the facts:

- KPI numbers,
- attendance rates,
- missing feedback counts,
- at-risk counts and anonymous IDs.

Gemini should only write the narrative:

- executive summary,
- key observations,
- risks,
- recommended next actions.

Gemini should not be trusted to calculate raw metrics directly from unprocessed sheet data.
