# 7. Pair Scores Auto Matching

## Plain-English Goal

Automatically match mentees to mentors using Gemini-assisted pair scoring.

The system should compare every eligible mentee against every available mentor, save those comparisons, then select the best final mentor for each mentee.

Final result:

```text
One mentee -> one mentor
```

The team reviews after the system creates the matches.

## Why This Is Separate From Mentee Scoring

Mentee scoring answers:

```text
How strong or ready is this mentee?
```

Pair scoring answers:

```text
How good is this specific mentor for this specific mentee?
```

Both are needed for matching.

## Sheets Used

| Sheet | Purpose |
| --- | --- |
| `Mentee Scores` | Decides which mentees are eligible for matching. |
| `Mentor Source Snapshot` | Gives mentor expertise, role, availability, and capacity. |
| `Pair Scores` | Stores every Gemini comparison between one mentee and one mentor. |
| `Match Recommendations` | Stores the best selected mentor per mentee. |
| `Matched Pairs` | Stores the final active assignments. |

## Matching Rule

Eligible mentees:

- `Final Score >= 60`
- `Gemini Review Status = Can Pair`
- has `Mentee ID`
- has `Mentee Email`

Available mentors:

- has `Mentor ID`
- has mentor name/email
- has remaining flexible capacity

Flexible capacity:

```text
Mentor stated capacity + 2
```

## Pair Score Breakdown

Gemini scores each pair using:

| Category | Weight |
| --- | ---: |
| Skill fit | 40 |
| Career fit | 30 |
| Availability fit | 15 |
| Capacity fit | 15 |
| Total | 100 |

## Pair Scores Sheet

Each row is one comparison.

Example:

| Mentee | Mentor | Total Pair Score |
| --- | --- | ---: |
| Deborah | Israel | 87 |
| Deborah | Jerry | 77 |
| Deborah | Rume | 65 |

Deborah is still assigned to only one mentor. The other rows are comparison evidence.

## Buttons To Build

### `Generate next pair score`

Scores one unscored mentee/mentor pair with Gemini and saves it in `Pair Scores`.

It must:

- skip pairs already scored,
- stop safely when Gemini quota is reached,
- preserve completed pair scores,
- allow resume later.
- save a clear `Gemini Concern` message if a pair fails, so the team can retry or troubleshoot.

This first implementation is the first sprint slice. It does not create final matches yet.

Test it by:

1. running `YDP Matching > Setup matching workbook`,
2. confirming `Pair Scores` exists,
3. running `YDP Matching > Generate next pair score`,
4. confirming one row appears in `Pair Scores`.

If the row status is `Error`, that row can be retried. The system only skips rows with `Pair Score Status = Scored`.

### `Generate pair scores batch`

Scores up to 5 unscored mentee/mentor pairs in one run.

This is the practical working button after the first one-pair test has succeeded.

It must:

- use the same Gemini pair scoring logic as `Generate next pair score`,
- preserve every existing `Pair Scores` row,
- skip only rows with `Pair Score Status = Scored`,
- retry rows marked `Error`,
- stop safely when Gemini quota is reached,
- stop safely if Apps Script is close to its run limit.

In layman terms: this button does not finish the entire program in one click. It does a small safe batch, saves the progress, and lets the team run it again later.

### `Auto-match from pair scores`

Uses saved pair scores to create final matches.

It should:

1. sort mentees by `Final Score` highest first,
2. find the best scored mentor for each mentee,
3. respect flexible capacity,
4. write the selected result to `Match Recommendations`,
5. write the final assignment to `Matched Pairs`,
6. mark matches as `Auto-Matched`,
7. mark review as `Pending Review`.

## Important Constraint

This can create many Gemini calls.

Example:

```text
50 mentees x 10 mentors = 500 pair scores
```

So pair scoring must be one-at-a-time/resumable, just like mentee scoring.

## Not In Version 1

- Sending match emails.
- WhatsApp messages.
- Web app display.
- Manual matching interface.
- Automatically changing mentor capacity after emails.
