# 4. YDP Program Control Center Tech Stack

## Purpose

Build an internal Vercel website where the YDP team can track automations, documentation, links, statuses, decisions, and build tasks from one place.

## Version 1 Stack

| Layer | Tool | Why |
| --- | --- | --- |
| Hosting | Vercel | Simple deployment for an internal web portal. |
| Web framework | Next.js | Works naturally on Vercel and supports protected routes, server-side data loading, and API routes. |
| Authentication | Google login | Team members can use their own Google accounts. |
| Access control | Vercel environment variable allowlist | Simple and secure for version 1. |
| Live tracker backend | Google Sheets | Non-technical team members can update statuses, owners, priorities, tasks, and links. |
| Documentation source | GitHub Markdown artifacts | Keeps detailed docs version-controlled. |
| Website mode | View-only first | Reduces risk and keeps Google Sheets as the editing surface. |
| Deployment secrets | Vercel environment variables | Stores allowed emails and API credentials safely. |
| Version control | GitHub | Tracks portal code and documentation. |

## Recommended App Shape

| Area | Purpose |
| --- | --- |
| Login page | Allows internal users to sign in with Google. |
| Home dashboard | Shows automation counts by status. |
| Automations table | Shows each automation, status, owner, priority, next action, and links. |
| Tasks table | Shows action items and build tasks. |
| Documentation library | Shows links to superpower artifacts and architecture docs. |
| Quick links | Shows Google Sheets, Apps Script projects, dashboards, and GitHub repo links. |
| Open decisions | Shows decisions still waiting for answers. |

## Planned Data Sources

| Source | Use |
| --- | --- |
| Google Sheet `Automations` tab | Main status tracker. |
| Google Sheet `Tasks` tab | Action item tracker. |
| Google Sheet `Decisions` tab | Key decision log. |
| Google Sheet `Links` tab | Central link library. |
| GitHub Markdown files | Detailed documentation and design notes. |

## Access Model

Version 1:

```text
Google login + ALLOWED_EMAILS environment variable
```

Later:

- Move allowlist into Google Sheets if the team needs non-technical access management.
- Add role-based permissions only if editing is added to the portal.

## Do Not Add In Version 1

- Website-based editing.
- Public pages.
- Complex admin panel.
- Role-based permissions.
- Database.
- Automatic syncing from every Apps Script project.

## Open Decisions Before Build

- Exact tracker sheet columns.
- First allowed user emails.
- Whether to render Markdown content inside the portal or link to GitHub.
- Whether version 1 needs charts or only tables.
- Repository/folder structure for the Vercel app.
