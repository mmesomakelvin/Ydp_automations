# 7. Pair Scores Auto Matching Tech Stack

## Purpose

Build the system-selected matching layer for the YDP Mentorship Program.

This layer compares eligible mentees against available mentors, stores Gemini pair scores, then writes final auto-matched assignments.

## Version 1 Stack

| Layer | Tool | Why |
| --- | --- | --- |
| Matching workbook | Google Sheets | The team can inspect source snapshots, pair scores, recommendations, and final matches. |
| Automation runtime | Google Apps Script V8 | Already deployed in the matching workbook and can read/write all needed tabs. |
| AI pair scoring | Gemini API | Scores qualitative mentor/mentee fit and provides match reasoning. |
| Source mentees | `Mentee Scores` sheet | Contains Gemini-assisted mentee readiness score and review status. |
| Source mentors | `Mentor Source Snapshot` sheet | Contains mentor expertise, role, availability, and capacity. |
| Raw pair storage | `Pair Scores` sheet | Audit trail for every mentor/mentee comparison. |
| Decision output | `Match Recommendations` sheet | Stores selected best mentor per mentee. |
| Final output | `Matched Pairs` sheet | Official final auto-created assignments. |
| Version control | GitHub + clasp | Keeps Apps Script code and docs deployable. |

## Gemini Usage

Gemini is part of the match decision.

It scores:

- skill fit out of `40`,
- career fit out of `30`,
- availability fit out of `15`,
- capacity fit out of `15`.

The total pair score is out of `100`.

Because Gemini quota is limited, the script must score one pair per run and resume later.

## Capacity Rule

Flexible mentor capacity:

```text
mentor stated capacity + 2
```

The system should prefer mentors within stated capacity, but can use the extra two slots when needed.

## Build Rules

- Do not delete existing `Mentee Scores` rows.
- Do not overwrite source snapshots except through the existing sync button.
- Do not send final match emails in version 1.
- Do not assign a mentee to more than one mentor.
- Do not assign mentors above flexible capacity.
- Preserve every completed `Pair Scores` row.
