# 5. Mentor-Mentee Matching Criteria Tech Stack

## Purpose

Create an explainable matching workflow that scores mentee readiness and recommends mentor-mentee pairings using application data.

## Version 1 Stack

| Layer | Tool | Why |
| --- | --- | --- |
| Source documentation | Google Doc | Contains the original selection and pairing criteria. |
| Source data | Mentee and mentor Google Sheets | Application responses already live there. |
| Stable identity | `Mentee ID` and `Mentor ID` columns | Prevents confusion when names repeat or someone submits more than once. |
| Scoring workspace | Google Sheets | Lets the team inspect and adjust scores before final decisions. |
| Automation runtime | Google Apps Script V8 | Reads form data, creates scoring columns, and generates match recommendations. |
| Optional narrative/scoring assistance | Gemini API | Can help summarize free-text answers, but should not make final decisions alone. |
| Human review | Google Sheets review columns | The team should approve or reject recommended matches before finalizing. |
| Final output | `Matched Pairs` sheet | Official source of active program pairings. |
| Version control | GitHub | Tracks criteria, scripts, and architecture docs. |

## Recommended Workflow

1. Assign unique IDs to mentors and mentees.
2. Score mentees using the selection criteria.
3. Select mentees based on score and available mentor capacity.
4. Generate ranked mentor recommendations for each selected mentee.
5. Show reasons for each recommendation.
6. Let the program team approve or edit matches.
7. Write approved matches into `Matched Pairs`.

## Scoring Rules

Mentee selection should use:

| Criteria | Weight |
| --- | --- |
| Learning Commitment & Prior Effort | 40% |
| Community Engagement & Participation | 20% |
| Career Goals & Program Alignment | 20% |
| Soft Skills & Mindset | 20% |

Each category is scored from `0` to `5`.

Final score:

```text
Learning * 8 + Community * 4 + Career * 4 + Mindset * 4
```

## Matching Rules

Priority order:

1. Skills and career path alignment.
2. Mentor availability/capacity.
3. Experience level fit.
4. Industry relevance.
5. Learning preferences.
6. Time zone and meeting availability.

## Do Not Add In Version 1

- Fully automatic final matching without review.
- Matching based only on names or manual memory.
- Gemini-only scoring without human validation.
- Matching applicants who were not selected into the program.
- Over-assigning mentors beyond their stated capacity.

## Open Decisions Before Build

- Whether mentee scoring should be manual, rule-based, or Gemini-assisted.
- Exact mentee response columns for career path, skills, learning style, goals, and YDP engagement.
- Exact mentor response columns for expertise, current role, years of experience, availability, capacity, and preferences.
- Whether to create a separate `Mentee Scores` sheet.
- Whether to create a separate `Match Recommendations` sheet before `Matched Pairs`.
