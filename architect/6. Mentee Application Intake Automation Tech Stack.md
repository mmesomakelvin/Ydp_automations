# 6. Mentee Application Intake Automation Tech Stack

## Primary Stack

| Layer | Choice | Reason |
| --- | --- | --- |
| Data source | Existing mentee Google Form response spreadsheet | Intake happens where the form response first lands. |
| Automation | Google Apps Script on-form-submit trigger | Already used by the mentee registration email automation. |
| Email | `MailApp` | Existing acknowledgement email flow already uses Apps Script email. |
| Tracking | New columns on the existing response tab | Keeps first-pass review close to the application data. |
| Dictionary | `Intake Dictionary` Google Sheet tab | Gives the team a plain-English reference inside the workbook. |
| Version control | GitHub + clasp | Same deployment pattern as the existing mentee automation. |

## Why This Should Not Be A New Script Project

The existing mentee script already handles:

- form-submit trigger,
- acknowledgement email,
- stable mentee IDs,
- duplicate email detection,
- tracking columns.

Application intake should reuse that same trigger so the mentee form does not have multiple competing automations.

## Gemini Usage

Do not use Gemini for instant intake in version 1.

Reason: Gemini quota/rate limits can fail during form submit. Intake must always run reliably.

Gemini remains in the matching workbook for deeper selection scoring and match explanations.

## New Sheet Columns

The mentee response tab should gain:

- `Intake Score`
- `Intake Status`
- `Intake Recommendation`
- `Intake Notes`
- `Duplicate Match Type`
- `Intake Processed At`

## New Helper Tab

The script should create:

```text
Intake Dictionary
```

The tab should include:

- column definitions,
- status definitions,
- score breakdown,
- operating instructions,
- reminder that intake score is not final selection score.

## Build Rules

- Keep all intake logic in the mentee script project.
- Do not create a second mentee form-submit trigger.
- Extend the existing `handleYdpMenteeFormSubmit` flow.
- Do not expose intake score/status to applicants.
- Duplicate applicants should still receive the normal acknowledgement email.
- Backfill old rows with a manual menu button before relying on the trigger for new rows.

## Future Extension

The mentor version can be built later using the same pattern, but mentor intake should focus on mentor capacity/profile quality rather than a selection-style score.
