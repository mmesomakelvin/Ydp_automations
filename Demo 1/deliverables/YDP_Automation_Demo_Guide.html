From Manual Work to a Controlled Mentorship System

How YDP automated registration communication, participant identity, Gemini-assisted scoring, mentor matching, deployment, and audit controls.

What We Built and Why

Before

Manual acknowledgements, repeated checking, duplicate-email risk, and matching decisions spread across form answers and memory.

Now

Google Sheets is the operating surface. Apps Script handles repeatable work. Statuses, dates, IDs, and logs preserve the audit trail.

Principle

Automate repetition, keep decisions visible, and test one row before any bulk action.

Registration Flow

1. Prepare
Setup email tracking columns
Creates the statuses and sent-at dates.
2. Identify
Assign IDs and mark duplicates
One stable ID per unique email.
3. Automate
Install form submit trigger
Future registrations run automatically.
4. Test
Preview or selected send
Validate one row before scaling.
5. Scale
Unsent-only bulk action
Rows already marked SENT are skipped.

How the Matching Flow Works

1. Sync
Source snapshots
Copy latest mentee and mentor applications.
2. Score
Mentee Scores
Gemini explains evidence; Apps Script applies fixed weights.
3. Compare
Pair Scores
Every eligible mentee is compared with every mentor.
4. Select
Auto-match
Highest score wins within mentor capacity.
5. Notify
Matched Pairs
Preview one pair, send one pair, then bulk send.

Mentee scoring

40% learning commitment; 20% each for community, goals, and soft skills. Final score 60+ means Can Pair.

Pair scoring

40 skill fit; 30 career fit; 15 availability; 15 capacity. Total is out of 100.

Exact Matching Order

  1. Sync source snapshots from the two form workbooks.
  2. Test Gemini connection.
  3. Finish mentee scores; completed rows are skipped and ERROR rows are retried.
  4. Finish all required pair scores for each Can Pair mentee.
  5. Run auto-match and review Match Recommendations plus Matched Pairs.
  6. Preview one matched pair, send one selected pair, then use bulk sending only after the test.

GitHub and clasp

What each tool does

clasp sends local Apps Script files to the correct Google project.

Git records each verified change.

GitHub keeps the shared source and history.

# Mentee: repository root clasp push --force # Mentor cd mentor clasp push --force # Matching cd matching clasp push --force # Save verified work git status --short git add <files> git commit -m "Describe change" git push origin main
DoEdit the correct local file, run tests, push with clasp from the correct folder, commit, then push to GitHub.
SafetyEach folder has its own .clasp.json. Running clasp from the wrong folder can update the wrong Apps Script project.

Safe Demo Rules

Do not expose

  • Gemini API key values
  • Participant phone numbers
  • Private application answers
  • Participant emails on a shared screen

Do not bulk send

  • Preview first.
  • Test one selected row.
  • Confirm SENT statuses and dates.
  • Bulk send only after approval.
